# Resume Work from Handoff

## Purpose

Load a handoff file from a previous session to restore the work context and continue from where it was left off.

## Prerequisites

- User must provide a handoff file path
- Handoff file must follow the standard format (generated by the handoff skill)

## Workflow

1. **Parse File Path**:
   - Check the path provided in args
   - If absolute path: use as-is
   - If relative path: resolve relative to workspace root
   - If passed via @ file reference: already converted to absolute path
2. **Read Handoff File**: Read the handoff file at the processed path
3. **Analyze Context**:
   - Task Overview: Understand the work goal
   - Current Progress: Check completed and in-progress tasks
   - Identified Issues: Review discovered problems
   - Key File Modifications: Check file modification status
   - Notes: What worked and what problems occurred
   - Next Steps: What needs to be done next
4. **Summarize Status**: Provide the user with a summary of current state
5. **Propose Next Action**: Ask if they want to start the Next Steps immediately
6. **Resume Work**: Execute Next Steps tasks after user approval

## Usage

### Method 1: Using @ File Reference (Recommended)

Type `@` in VSCode to trigger the file selection UI:

```bash
/load_handoff @HANDOFF-2026-02-10-143045.md
```

**Benefits**:
- Easy file selection via UI
- Filename auto-completion support
- Automatically converted to absolute path

### Method 2: Using Relative Path

```bash
/load_handoff .claude/handoffs/task-name/HANDOFF-2026-02-10-143045.md
```

Uses a path relative to the workspace root.

### Method 3: Using Absolute Path

```bash
/load_handoff /home/user/workspace/.claude/handoffs/task-name/HANDOFF-2026-02-10-143045.md
```

### Skill Invocation

```
skill: "load-handoff"
args: ".claude/handoffs/knowledge-system-improvement/HANDOFF-2026-02-10-143045.md"

Or when passed via @ reference:

```
skill: "load-handoff"

After reading the handoff file, verify:

## Resume Work Workflow

1. **Load Context**: Read handoff file
2. **Summarize Status**: Provide current situation summary to user
3. **Request Approval**: Ask if Next Steps should be started
4. **Execute Work**:
   - Perform the first Next Steps task
   - Read, modify, or create files as needed
   - Update status after completion
5. **Create New Handoff** (optional): If additional work remains
   - Update progress
   - Write new Next Steps

## Best Practices

- **Use @ File Reference**: Leverage VSCode's file selection UI for convenience
- **Maintain Context**: Be careful not to lose the context of previous work
- **Incremental Progress**: Perform only one Next Steps task at a time
- **Clear Communication**: Clearly explain what work is being done
- **Track Status**: Mark completed tasks with ✅ and update progress

## Notes

- Handoff files are saved in the format `.claude/handoffs/[task-slug]/HANDOFF-YYYY-MM-DD-HHMMSS.md` (project root's `.claude/` folder)
- When multiple handoff files exist, use the most recent one
- **NEVER** save in `~/.claude/` (global user folder) — only use the project root's `.claude/handoffs/` directory
- After resuming work, if there's additional progress, create a new handoff file to maintain continuity
- When using @ file reference, it's automatically converted to an absolute path before being passed to the skill
